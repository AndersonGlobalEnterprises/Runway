# Brand Brains — the soul of every Trimwind client

**What this is:** every Trimwind client gets ONE file here — their Brand Brain. It is the single source of truth every content agent loads BEFORE writing, reviewing, or rendering anything for that client. If a draft conflicts with the client's Brand Brain, the Brand Brain wins.

**Why it exists:** this is Trimwind's moat. Generic AI content sounds generic because the model has no doctrine. A Brand Brain gives each client a locked voice, a visual wavelength, a proof inventory (real claims only), a mechanism guard, and a Pre-Flight Checklist that grounds anything weak. The file gets smarter every month — leaving Trimwind means abandoning it.

| File | What it is |

| `AGE.md` | **Client #0 — Anderson Global Enterprises.** Trimwind runs AGE's own content first. This is the proof. |
| `_TEMPLATE.md` | The onboarding template. Copy to `<client-slug>.md` for each new client and fill from their REAL materials (site, reviews, past posts — Firecrawl them; never invent). |

## The law (for every agent in this repo)

1. **Load first.** `runway-creative` and `runway-review` read the client's Brand Brain before touching a flight. No Brand Brain on file → generate nothing; flag it.
2. **Pre-Flight gates everything.** Any single "no" on the client's Pre-Flight Checklist = the flight is grounded. No exceptions for volume or deadlines.
3. **Proof is strict.** Only claims listed in the client's Proof Inventory (or freshly sourced with a URL + date) may ship. A fabricated metric is a fireable offense.
4. **Mechanism guard.** Teach the client's mindset, never reveal their (or AGE's) build/pipeline/secret.
5. **Update, don't fork.** Learnings from performance go INTO the client's Brand Brain (dated, in §Learnings). One file per client, forever.
